# PhysLab AI

**AI-платформа сопровождения учебного физического исследования**

PhysLab AI помогает учащемуся пройти полный цикл лабораторного исследования: сформулировать гипотезу, провести эксперимент, проверить качество данных, сопоставить результат с физической моделью, получить диагностическую гипотезу ML-модели, выбрать следующее действие и сформулировать собственный вывод.

Проект не заменяет рассуждение готовым ответом. ИИ используется как инструмент сопровождения исследования.

---

## Конкурсная категория

Проект подготовлен для направления **«AI в образовании»** конкурса JUNIOR ML CONTEST.

PhysLab AI представляет собой AI-first образовательный продукт, в котором машинное обучение используется для анализа лабораторных данных, выявления возможных ошибок эксперимента и поддержки самостоятельных исследовательских решений учащегося.

---

## Проблема

Школьная лабораторная работа часто воспринимается как выполнение инструкции по заранее известному алгоритму:

1. повторить действия;
2. записать измерения;
3. подставить значения в формулу;
4. получить ожидаемый результат.

В таком формате учащийся редко принимает самостоятельные решения и не всегда понимает:

- почему эксперимент отклонился от теоретической модели;
- насколько надёжны измерения;
- нужно ли повторить отдельную точку или всю серию;
- какая физическая причина могла повлиять на результат;
- можно ли соглашаться с автоматической рекомендацией.

PhysLab AI возвращает в лабораторную работу исследовательскую логику.

## Целевая аудитория

### Основная

- учащиеся старших классов;
- студенты первых курсов, выполняющие базовый физический практикум;
- преподаватели физики;
- организаторы школьных и университетских лабораторий.

### Пользовательские задачи

**Учащийся:**

- проверить качество экспериментальных данных;
- понять возможную причину отклонения;
- выбрать следующее исследовательское действие;
- сопоставить данные с физической моделью;
- сформулировать собственный вывод;
- увидеть профиль проявленных исследовательских действий.

**Преподаватель:**

- быстрее выявить типичные ошибки эксперимента;
- получить прозрачное объяснение ML-диагностики;
- организовать обсуждение данных вместо выдачи готового ответа;
- оценить действия учащегося по структурированной рубрике.

---

## Поддерживаемые лабораторные работы

1. **Нагревание и охлаждение**
   - анализ температурной зависимости во времени;
   - сопоставление с законом охлаждения Ньютона;
   - выявление выбросов, шума и дрейфа датчика.

2. **Закон Бойля — Мариотта**
   - анализ зависимости давления от объёма;
   - проверка условия `pV = const`;
   - выявление утечки газа, изменения температуры и ошибок измерения объёма.

3. **Изохорный процесс**
   - анализ зависимости давления от абсолютной температуры;
   - проверка условия `p/T = const`;
   - выявление нестабильности объёма и запаздывания температурного датчика.

4. **Тепловой баланс**
   - анализ процесса установления теплового равновесия;
   - оценка теплопотерь;
   - выявление недостаточного перемешивания и ошибок измерения массы.

## Педагогическая логика

PhysLab AI не сообщает учащемуся «правильный ответ».

После анализа данных платформа предлагает:

- диагностическую гипотезу;
- признаки, на которые опиралась модель;
- вопрос для самостоятельного рассуждения;
- следующее экспериментальное действие;
- альтернативные гипотезы.

Итоговая полярная диаграмма строится не по длине или наличию текста, а по результатам структурированных заданий. Каждое задание связано с конкретным исследовательским действием и оценивается по шкале от 0 до 3.

Открытые ответы сохраняются для рефлексии учащегося и анализа преподавателем, но не повышают балл автоматически.

## Data Science

### Стратегия данных

Проект использует два независимых контура данных.

#### 1. Физически обусловленные синтетические данные

Синтетические эксперименты моделируют:

- идеальную физическую зависимость;
- поведение измерительного оборудования;
- шум и дрейф датчиков;
- ошибки проведения эксперимента;
- вариативность условий и параметров установки.

Синтетические данные используются для обучения и количественной оценки моделей.

#### 2. Реальные внешние данные

Реальные данные используются для:

- проверки физических диапазонов;
- оценки характерного уровня шума;
- калибровки генераторов;
- проверки переносимости пайплайна;
- тестирования загрузки и нормализации;
- внешнего анализа поведения моделей.

Реальные серии без достоверной экспертной разметки не используются как псевдоразмеченные обучающие примеры.

### Текущий корпус реальных данных

- исходных файлов: **12**;
- обнаруженных таблиц: **23**;
- сформированных нормализованных экспериментов: **36**;
- экспериментов, готовых для моделей: **35**;
- обработанных моделью реальных серий: **35**;
- профилей температурных датчиков: **9**.

### Валидация

Для оценки моделей используется групповая отложенная выборка. Эксперименты, полученные из одной генерационной группы, не разделяются между обучением и проверкой.

Основная метрика:

- **Macro F1**.

Дополнительные метрики:

- Accuracy;
- Balanced Accuracy;
- Precision и Recall по классам;
- матрица ошибок;
- среднее и стандартное отклонение кросс-валидации;
- разрыв между обучением и независимой проверкой.

### Текущие результаты

| Лабораторная работа | Macro F1 |
|---|---:|
| Нагревание и охлаждение | 0.783 |
| Закон Бойля — Мариотта | 0.956 |
| Изохорный процесс | 0.981 |
| Тепловой баланс | 0.893 |

Метрики рассчитаны на независимой групповой синтетической holdout-выборке. Они не заявляются как точность на реальных учащихся.

### Подбор гиперпараметров

В проекте реализован `GridSearchCV` с групповой валидацией. Настроенная модель заменяет текущую только при подтверждённом улучшении на независимой holdout-выборке.

По результатам последней проверки текущие откалиброванные модели были сохранены, поскольку кандидаты Grid Search не дали устойчивого улучшения.

## Воспроизводимость

Проект использует:

- фиксированные случайные состояния;

---

## Ограничения

1. Большая часть обучающего корпуса является синтетической.
2. Реальные внешние серии преимущественно не имеют экспертной разметки классов ошибок.
3. Текущие метрики нельзя интерпретировать как доказанную эффективность на реальных учащихся.
4. Профиль исследовательских действий описывает одну лабораторную работу и не является психологической диагностикой.
5. ML-рекомендация является гипотезой и требует проверки учащимся и преподавателем.
6. Для доказательства образовательного эффекта необходим пилот с учащимися и контрольными измерениями.

- ИИ не выдаёт готовый вывод за учащегося;
- рекомендация обозначается как диагностическая гипотеза;
- исходные реальные данные не изменяются;
- неразмеченные реальные данные не используются как подтверждённые ответы;
- метрики и ограничения показываются открыто;
- окончательное решение остаётся за человеком;
- образовательный профиль не используется как психологический диагноз.
