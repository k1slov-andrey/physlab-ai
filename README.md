# PhysLab AI

**Интеллектуальная платформа для цифрового физического практикума**

PhysLab AI анализирует экспериментальные данные, сопровождает учащегося от гипотезы до доказательного вывода и помогает преподавателю видеть исследовательскую траекторию каждого.

**Статус:** рабочий MVP

## Зачем создан проект

Цифровые лаборатории ускоряют измерения, формируют таблицы и строят графики, но сами по себе не превращают выполнение инструкции в исследование. Учащийся может получить корректные данные и при этом не понять, почему результат отклонился от теории, какое условие было нарушено и насколько обоснован итоговый вывод.

PhysLab AI добавляет к цифровому эксперименту интеллектуальное сопровождение. Платформа помогает проверить качество данных, сопоставить эксперимент с физической моделью, сформировать диагностическую гипотезу о возможной ошибке и выбрать следующее действие. Итоговое решение остаётся за учащимся и преподавателем.

Проект ориентирован на профильные технологические, инженерные, физико-математические и естественно-научные классы, школьные технопарки, «Кванториумы», центры «Точка роста», дополнительное образование и первые курсы технических и естественно-научных программ.

## Что уже работает

- четыре лабораторных ML-модуля;
- демонстрационные физически обусловленные эксперименты;
- загрузка файлов `CSV`, `XLSX` и `XLS`;
- автоматическое определение лабораторной работы;
- сопоставление столбцов и преобразование единиц;
- проверка пропусков, дубликатов и физических диапазонов;
- сравнение экспериментальной и модельной кривых;
- расчёт `RMSE`, `MAE`, `R²` и максимального отклонения;
- классификация возможной экспериментальной ошибки;
- объяснение диагностической гипотезы;
- исследовательский вопрос и рекомендуемое действие;
- структурированная педагогическая диагностика;
- профиль исследовательских действий;
- экспорт нормализованных данных;
- работа с внешним корпусом реальных измерений.

## Лабораторные модули

### Нагревание и охлаждение

Анализ зависимости температуры от времени, сопоставление с законом охлаждения Ньютона, выявление выбросов, шума, дрейфа и инерционности датчика.

### Закон Бойля — Мариотта

Анализ зависимости давления от объёма, проверка условия `pV = const`, выявление утечки газа, изменения температуры и ошибок измерения объёма.

### Изохорный процесс

Анализ зависимости давления от абсолютной температуры, проверка условия `p/T = const`, выявление нестабильности объёма и запаздывания температурного датчика.

### Тепловой баланс

Анализ установления теплового равновесия, оценка теплопотерь, выявление недостаточного перемешивания и ошибок измерения массы или температуры.

## Как устроен анализ

```text
CSV / Excel
        ↓
проверка и нормализация
        ↓
физически интерпретируемые признаки
        ↓
специализированная ML-модель
        ↓
сопоставление с физическим законом
        ↓
диагностическая гипотеза
        ↓
следующее действие учащегося
```

Модель не выдаёт окончательный ответ. Она формирует проверяемую гипотезу, показывает основания прогноза и предлагает исследовательский вопрос. Учащийся может согласиться с рекомендацией, оспорить её или провести дополнительное измерение.

Полярная диаграмма строится по результатам структурированных заданий и отражает действия, проявленные в конкретной лабораторной работе. Она не является психологической характеристикой или автоматической итоговой оценкой.

## Данные и результаты моделей

Для обучения используются физически обусловленные синтетические данные с управляемыми сценариями ошибок. Реальные внешние данные применяются для проверки загрузки и нормализации, оценки шумов, калибровки генераторов и анализа переносимости пайплайна.

Текущий внешний корпус включает:

- 12 исходных файлов;
- 23 обнаруженные таблицы;
- 36 нормализованных экспериментов;
- 35 серий, пригодных для обработки текущими моделями;
- 9 профилей температурных датчиков.

Модели оценивались на независимой групповой синтетической holdout-выборке. Эксперименты из одной генерационной группы не разделялись между обучением и проверкой.

| Лабораторная работа | Accuracy | Balanced Accuracy | Macro F1 |
|---|---:|---:|---:|
| Нагревание и охлаждение | 0,781 | 0,781 | 0,783 |
| Закон Бойля — Мариотта | 0,956 | 0,956 | 0,956 |
| Изохорный процесс | 0,981 | 0,981 | 0,981 |
| Тепловой баланс | 0,894 | 0,894 | 0,893 |

`Macro F1` используется как основная метрика, поскольку каждый класс экспериментальной ошибки одинаково значим. Совпадение `Accuracy` и `Balanced Accuracy` связано со сбалансированным представительством классов в holdout-выборке.

Эти значения характеризуют способность моделей различать заложенные сценарии генерации. Они не интерпретируются как доказанная точность на реальных учащихся.

## Быстрый запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/k1slov-andrey/physlab-ai.git
cd physlab-ai
```

### 2. Создание виртуального окружения

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установка зависимостей

```bash
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

### 4. Запуск приложения

```bash
python -m streamlit run app.py
```

После запуска приложение доступно по адресу:

```text
http://localhost:8501
```

## Автоматические проверки

Установка зависимостей для разработки:

```bash
python -m pip install -r requirements-dev.txt
```

Запуск тестов:

```bash
python -m pytest
```

В проекте реализованы 16 автоматических тестов и GitHub Actions CI. Проверка включает установку зависимостей, запуск тестов, сборку Docker-образа, запуск контейнера и проверку доступности Streamlit-приложения.

## Ограничения

1. Основная часть обучающего корпуса является синтетической.
2. Реальные внешние серии преимущественно не имеют экспертной разметки классов ошибок.
3. Текущие метрики не являются доказанной эффективностью на реальных учащихся.
4. Профиль исследовательских действий описывает одну лабораторную работу и не является общей оценкой личности.
5. Диагностическая гипотеза модели требует проверки учащимся и преподавателем.

Следующий этап проекта — сбор экспертно размеченных данных, кабинет преподавателя, матрица класса и полевая апробация в реальном образовательном процессе.
