# PhysLab AI

PhysLab AI — локальное приложение для анализа данных школьного физического эксперимента. Система проверяет структуру измерений, рассчитывает физически интерпретируемые признаки, формирует диагностическую гипотезу и предлагает следующий шаг. Итоговый вывод остаётся за учащимся и преподавателем.

## Задача

Цифровая лаборатория ускоряет измерения и построение графиков, но не объясняет, почему серия отклонилась от физической модели и какое действие следует выполнить дальше. При работе с классом преподавателю приходится одновременно контролировать оборудование, проверять данные и разбирать ход рассуждения каждого учащегося.

PhysLab AI автоматизирует первичную проверку экспериментальной серии и сохраняет преподавателя в контуре принятия решения. Система предназначена для учебного практикума, а не для выставления итоговой оценки без участия педагога.

## Что реализовано

- четыре лабораторных ML-модуля;
- загрузка `CSV`, `XLSX` и `XLS`;
- проверка структуры файла, пропусков, дубликатов и физических диапазонов;
- сопоставление названий столбцов и преобразование единиц;
- построение экспериментальной и расчётной кривых;
- расчёт `RMSE`, `MAE`, `R²` и максимального отклонения;
- классификация типового сценария ошибки;
- проверка надёжности прогноза и отказ с классом `unknown`;
- диагностическая гипотеза, исследовательский вопрос и рекомендуемое действие;
- структурированный профиль исследовательских действий;
- Streamlit-интерфейс, Docker и GitHub Actions;
- единая команда локальной проверки проекта.

## Лабораторные модули

| Модуль | Проверяемая зависимость | Классы |
|---|---|---|
| Нагревание и охлаждение | температура от времени | `normal`, `single_outlier`, `sensor_drift`, `high_noise` |
| Закон Бойля — Мариотта | `pV = const` | `normal`, `air_leak`, `temperature_change`, `volume_measurement_error` |
| Изохорный процесс | `p/T = const` | `normal`, `air_leak`, `volume_instability`, `temperature_sensor_lag` |
| Тепловой баланс | установление равновесия | `normal`, `heat_loss`, `mass_measurement_error`, `insufficient_mixing` |

## Контур анализа

```text
CSV / Excel
        ↓
валидация и нормализация
        ↓
физически интерпретируемые признаки
        ↓
ML-модель конкретной лабораторной работы
        ↓
проверка надёжности прогноза
        ↓
диагностическая гипотеза или unknown
        ↓
решение учащегося и проверка преподавателя
```

Если входная серия выходит за область обучающих признаков, вероятность классов недостаточно различима или максимальная вероятность мала, система не принуждает модель к одному из известных классов. В ответе сохраняется кандидат, но итоговый статус становится `unknown` с указанием причины отказа.

## Данные и протокол оценки

Обучающий контур построен на физически обусловленных генераторах. Для каждой лабораторной работы формируется 160 экспериментальных семейств. Одно семейство содержит четыре варианта с одинаковыми условиями измерения и разными классами ошибки.

Для каждого модуля используется 640 серий:

- `train`: 320 серий, 80 семейств;
- `validation`: 160 серий, 40 семейств;
- `test`: 160 серий, 40 семейств.

Семейства не пересекаются между частями. Базовая и настроенная конфигурации сравниваются по групповой кросс-валидации на development-части. Сохранённая `test`-часть используется только для итоговой оценки установленной модели.

### Итоговые показатели установленных моделей

Источник: [`evaluation/final_model_summary.csv`](evaluation/final_model_summary.csv).

| Лабораторная работа | Модель | Accuracy | Balanced Accuracy | Macro F1 |
|---|---|---:|---:|---:|
| Нагревание и охлаждение | Random Forest | 0,875 | 0,875 | 0,876 |
| Закон Бойля — Мариотта | Random Forest | 0,925 | 0,925 | 0,925 |
| Изохорный процесс | Random Forest | 0,994 | 0,994 | 0,994 |
| Тепловой баланс | Gradient Boosting | 0,888 | 0,888 | 0,887 |

`Macro F1` выбрана основной метрикой, поскольку все четыре класса в каждой задаче имеют одинаковую значимость. Эти показатели характеризуют распознавание сценариев генератора на ранее не встречавшихся экспериментальных семействах. Они не являются оценкой качества на реальных учащихся или неизвестном оборудовании.

## Реальные файлы

В репозитории сохранены результаты ранее проведённого аудита внешнего корпуса: 12 файлов и 25 таблиц. Исходные файлы `real_data_raw` не публикуются в текущей версии, поэтому повторная строгая нормализация корпуса из чистого клона невозможна.

Код отбора физических каналов исправлен и покрыт тестами: давление, расход, влажность, масса, объём и другие нецелевые столбцы не принимаются за температуру. До повторного запуска на первичных файлах старые нормализованные серии и прогнозы рассматриваются только как технический журнал интеграции, а не как подтверждение качества моделей.

## Быстрый запуск

Требуется Python 3.11–3.13.

```bash
git clone https://github.com/k1slov-andrey/physlab-ai.git
cd physlab-ai
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python -m streamlit run app.py
```

Приложение откроется по адресу `http://localhost:8501`.

Для разработки установите дополнительные зависимости:


## Ограничения

1. Основной обучающий корпус синтетический.
2. Внешние серии не имеют полной экспертной разметки классов ошибок.
3. Отказ `unknown` снижает риск ложной уверенности, но не заменяет полевую OOD-валидацию.
4. Профиль исследовательских действий относится к одной работе и не является психологической характеристикой.
5. Образовательный эффект и экономия времени преподавателя ещё не подтверждены контролируемым пилотом.
